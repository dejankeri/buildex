# linear-search - read what's actually being worked on

Pull the relevant Linear issue or project into the conversation before you reason about status.

## When to use

- The operator asks "what's the status of…", "what's blocking…", "what's left in this cycle?".
- You are about to file or update work and need to avoid duplicating an existing issue.

## Steps

1. Search Linear for the issue, project, or label with the Linear tools; prefer recent activity.
2. Read the issue fully - state, assignee, comments - before summarizing.
3. If several issues match, list them briefly and ask which one rather than guessing.
4. Link the issue you used so the operator can open it.

## Rules

- Reading Linear is safe and runs freely; **creating or changing** issues waits for approval -
  never assume permission to file or reassign.
