# linear-issue - create or update a Linear issue

Turn a request into a well-formed Linear issue, proposed for the operator's approval.

## When to use

- The operator says "make an issue for…", "move that to done", "add this to the roadmap".

## Steps

1. Search first (see linear-search) to avoid duplicating an existing issue - update it if it exists.
2. Draft the issue: a clear title, a short description with the why, and the right team/project.
3. Propose the create/update; it is an outward change, so it surfaces as an approval card - wait.
4. After approval, share the issue link and note what you set.

## Rules

- Prefer updating an existing issue over filing a near-duplicate.
- Don't invent scope or assignees to fill fields; leave what you don't know for the operator.
