# intercom-reply - propose a support reply

Turn a request into a clear, on-brand Intercom reply, proposed for the operator's approval.

## When to use

- The operator says "reply to them", "close this", "let them know…".

## Steps

1. Read the full thread first (see intercom-inbox) so the reply answers what was actually asked.
2. Draft a concise, helpful response in the operator's voice; state any close/snooze you'd apply.
3. Propose it; a customer-facing message is outward, so it surfaces as an approval card - wait.
4. After approval, confirm it sent and note any status change.

## Rules

- Every customer-facing message is human-gated - never send, close, or snooze unattended.
- Don't promise fixes, refunds, or timelines the operator hasn't confirmed.
