# capture-decision - record a company decision

Every company decision that a future reader would otherwise have to reverse-engineer gets written
down in the same session it is made. This is how the brain accrues judgment, not just facts.

## When to use

- A choice was made about direction, pricing, hiring, a vendor, or a policy.
- A prior decision is being changed or reversed.
- A trade-off was weighed that isn't obvious from the outcome alone.

## Steps

1. Open `decisions/log.md` (create it if missing).
2. Add a dated entry at the top: **Decision**, **Context** (what forced it), **Alternatives**
   (what else was considered), **Consequence** (what it commits us to; what it supersedes).
3. If it changes a prior decision, say so explicitly - never edit the old entry away.

## Rules

- One decision per entry; dates are absolute.
- Plain markdown; nothing secret (the log is part of the portable brain).
- Superseding is honest: the new entry wins, the old one stays as history.
